# Implementation Plan - Fixing IDE "381 Problems"

The current high problem count (381) is primarily caused by the IDE (VS Code) attempting to lint **compiled Blade views** and **cached files** in the `storage` directory. These files are automatically generated by Laravel and often contain syntax that linters don't understand, or references to variables defined at runtime.

### 1. Configure VS Code Settings
I will create a `.vscode/settings.json` file to:
- Exclude the `storage/framework/views` directory from the PHP linter.
- Hide IDE helper files from the file explorer to reduce clutter while keeping them indexed.

### 2. Standardize View Directives
I've already refactored `@forelse` to `@if`/`@foreach`, but I will do a final sweep to ensure no trailing spaces or missing newlines are causing parser issues in the source Blade files.

### 3. Update Model Mixins
I will update the Models to use the specific `_ide_helper_models.php` format, which is more robust than manual annotations.

### 4. Refresh IDE Cache
I will provide instructions to the user on how to refresh their IDE index, as many of these "problems" refer to classes that have already been fixed but are still cached in the IDE's internal state.
